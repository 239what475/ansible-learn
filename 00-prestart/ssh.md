# SSH 初始化

这一节记录本机 `SSH` 密钥的初始化过程。

后面的 `Ansible` 教程默认通过 `SSH` 连接 `Incus` 容器，因此建议先准备好一对可用的密钥。

## 目标

完成这一节后，你应该具备下面这些条件：

- 本机存在可用的 `SSH` 密钥对
- 公钥内容可以方便地复制到目标节点
- 后续可以把公钥加入容器用户的 `~/.ssh/authorized_keys`

## 1. 准备 `~/.ssh` 目录

如果目录还不存在，先创建并设置权限：

```bash
mkdir -p ~/.ssh && chmod 700 ~/.ssh
```

### 说明

- `~/.ssh` 是当前用户默认的 `SSH` 配置目录。
- `700` 表示只有当前用户自己可以访问这个目录。
- 如果权限过宽，`SSH` 有时会拒绝使用相关文件。

## 2. 生成密钥对

生成一对 `ed25519` 密钥：

```bash
ssh-keygen -t ed25519 -C "$USER@$(hostname)"
```

执行过程中，通常会看到这些提示：

- 保存路径：默认直接回车即可，通常会生成到 `~/.ssh/id_ed25519`
- 密码短语：
  - 如果你希望更安全，可以设置 passphrase
  - 如果这是本地学习环境，也可以暂时留空

生成完成后，通常会得到两个文件：

- `~/.ssh/id_ed25519`：私钥，不要随意泄露
- `~/.ssh/id_ed25519.pub`：公钥，可以分发到目标机器

## 3. 查看公钥内容

输出公钥内容：

```bash
cat ~/.ssh/id_ed25519.pub
```

这一步的目的是把公钥复制出来，后续放到目标节点用户的：

- `~/.ssh/authorized_keys`

## 4. 建议做的验证

### 查看密钥文件是否生成

```bash
ls -l ~/.ssh/id_ed25519 ~/.ssh/id_ed25519.pub
```

### 检查目录权限

```bash
ls -ld ~/.ssh
```

你通常应该看到类似 `drwx------` 的权限，也就是 `700`。

## 5. 后续会怎么用

等我们开始配置 `Incus` 容器后，这把公钥会被放到容器用户的 `authorized_keys` 中。

然后 `Ansible` 就可以通过下面这种方式连接目标节点：

- 用户名：`ansible_user`
- 私钥：`~/.ssh/id_ed25519`

这也就是为什么 `01-quickstart/inventory.yml.example` 里会有：

- `ansible_ssh_private_key_file`

## 常见提醒

### 1) 私钥不要提交到 Git

下面这些文件都应该只保留在你本机：

- `~/.ssh/id_ed25519`
- `~/.ssh/id_ed25519.pub`

尤其是私钥，绝对不要放进仓库。

### 2) 公钥可以公开，私钥不可以

- `.pub` 文件是公钥，可以复制到目标机器
- 没有 `.pub` 后缀的是私钥，必须自己保管好

### 3) 学习环境和生产环境的安全要求不同

在学习环境中，你可以为了方便先不设置 passphrase；
但在正式环境中，一般建议给私钥加口令并妥善管理。
